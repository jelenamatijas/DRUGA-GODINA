Virtuelna okruenja (virtual environments)

Kako bismo izbjegli probleme s koritenjem globalnog Python interpretera, napraviemo virtuelno okruenje za projekat sa sopstvenom kopijom Python interpretera u koje se mogu instalirati Python moduli specifini za to okruenje, bez da imaju efekta na globalni Python interpreter.

Jedan od naina u VSCode je komanda >Python: Create Environment... (moe se ukucati kroz pretragu sa F1). Moe se koristiti venv opcija. Kreiranje terminala u VSCode sa uitanim varijablama virtuelnog okruenja moe se vriti komandom >Python: Create Terminal. Na Windows-u, sa komandom na terminalu where.exe python se moe provjeriti da je korektno podeena putanja na Python interpreter specifian za kreirano virtuelno okruenje (potfolder projekta, .venv/Scripts). Potrebno je i odabrati Jupyter kernel koji odgovara Python interpreteru iz virtuelnog okruenja.

Bez upotrebe VSCode komandi, venv virtuelno okruenje se moe kreirati komandom python -m venv .venv. Varijable virtuelnog okruenja se mogu uitati komandom .\venv\Scripts\activate.bat u CMD-u, odnosno .\.venv\Scripts\activate.ps1 u PowerShell-u. U sluaju da se ne moe izvriti PowerShell skripta zbog restrikcija, moe se iskoristiti komanda Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope CurrentUser.

import numpy as np

def multiply_matrices(A, B):
    rows = A.shape[0]
    cols = B.shape[1]

    if A.shape[1] != B.shape[0]:
        raise ValueError("Number of columns in A must be equal to number of rows in B")
    
    C = np.zeros((rows, cols))

    for i in range(rows):
        for j in range(cols):
            for k in range(A.shape[1]):
                C[i, j] += A[i, k] * B[k, j]

    return C

import time

# Definiemo funkciju koja mjeri vrijeme izvravanja funkcije.
def measure_runtime(func, *args, warmup_rounds=0, measure_rounds=1):
    '''
    Measure the runtime of a function.

    Parameters
    ----------
    func : callable
        The function to measure.
    args : tuple
        The arguments to call the function with.
    warmup_rounds : int
        The number of warmup rounds to run before the actual measurement.
    measure_rounds : int
        The number of measurement rounds to run.
    '''

    # Runde zagrijavanja.
    # Korisno za funkcije koje se izvravaju bre nakon prvih nekoliko poziva.
    # Razlog je to se u prvim pozivima funkcije i podaci uitavaju u glavnu memoriju, odnosno u ke memoriju.
    for _ in range(warmup_rounds):
        func(*args)

    total_time = 0.0
    
    # Mjerimo vrijeme izvravanja funkcije vie puta, a zatim raunamo prosjeno vrijeme izvravanja.
    # Ovo radimo kako bismo smanjili uticaj sluajnih faktora na mjerenje.
    # Na primjer, ukoliko je CPU zauzet nekim drugim zadatkom, vrijeme izvravanja funkcije moe biti vee.
    # Mjerenjem vie puta, smanjujemo uticaj ovakvih faktora.
    # Tj. na ovaj nain smanjujemo um u mjerenju i dobijamo tanije rezultate mjerenja.
    for _ in range(measure_rounds):
        start_time = time.time()
        func(*args)
        end_time = time.time()
        total_time += end_time - start_time

    average_time = total_time / measure_rounds
        
    return average_time

# Funkcija za mjerenje vremena izvravanja mnoenja matrica za vie veliina.
def run_measurements(sizes, *, warmup_rounds=2, measure_rounds=5):
    """
    Run performance measurements for different matrix sizes.
    
    Parameters:
    sizes : list - List of matrix sizes to test
    warmup_rounds : int - Number of warmup rounds
    measure_rounds : int - Number of measurement rounds
    
    Returns:
    manual_times, numpy_times : tuple - Lists of execution times
    """
    manual_times = []
    numpy_times = []
    
    for size in sizes:
        print(f"Measuring for size {size}x{size}...")
        A = np.random.rand(size, size)
        B = np.random.rand(size, size)
        manual_time = measure_runtime(multiply_matrices, A, B, 
                                warmup_rounds=warmup_rounds, 
                                measure_rounds=measure_rounds)
        numpy_time = measure_runtime(np.matmul, A, B, 
                                warmup_rounds=warmup_rounds, 
                                measure_rounds=measure_rounds)
        manual_times.append(manual_time)
        numpy_times.append(numpy_time)
        
    return manual_times, numpy_times

Matematiki model za vrijeme izraunavanja

Broj koraka izraunavanja prethodnog algoritma moemo predstaviti kao

$$t(n) = K_3n^3 + K_2n^2 + K_1n + K_0 $$

Ako pretpostavimo $K_3n^3 \gg K_2n^2 + K_1n + K_0$, onda

$$t(n) \approx K_3n^3 \implies K_3 \approx \frac{t(n)}{n^3}\text{.}$$

Konstruisaemo grafik na kom su prikazana mjerenja stvarnog vremena izraunavanja, kao i grafik opisanog matematikog modela vremena izraunavnja, za koji emo, na osnovu mjerenja, procijeniti koeficijent $K$.

elimo da utvrdimo koliko dobro matematiki model opisuje stvarno vrijeme izvravanja. U tu svrhu emo iskoristiti samo dio mjerenja za estimaciju koeficijenta, pa emo onda vizuelno odrediti koliko matematiki model dobro predvia vrijeme izvravanja, poredei ga sa preostalim mjerenjima.

In[9]:
import numpy as np

# Vrimo procjenu koeficijenta K_3
manual_coeffs = [manual_times[i] / (sizes[i]**3) for i in range(len(sizes))]

# Koristimo koeficijent iz sredine opsega
start = int(len(manual_coeffs) * 0.3)
end = int(len(manual_coeffs) * 0.7)
k_manual = sum(manual_coeffs[start:end]) / len(manual_coeffs[start:end])

# Procjenjujemo koeficijent K_3 za NumPy
numpy_coeffs = [numpy_times[i] / (sizes[i]**3) for i in range(len(sizes))]
k_numpy = sum(numpy_coeffs[start:end]) / len(numpy_coeffs[start:end])

# Generiemo take za matematiki model
model_sizes = np.linspace(min(sizes), max(sizes), 100)
model_times_manual = [k_manual * (n**3) for n in model_sizes]
model_times_numpy = [k_numpy * (n**3) for n in model_sizes]

# Plotujemo grafik za mjerenja vs. matematiki model
plt.figure(figsize=(10, 6))
plt.plot(sizes, [t*1000 for t in manual_times], 'o', label='Manual (measured)', markersize=8)
plt.plot(model_sizes, [t*1000 for t in model_times_manual], '--', 
         label=f'Manual model: {k_manual:.2e}n', linewidth=2)
plt.plot(sizes, [t*1000 for t in numpy_times], 'o', label='NumPy (measured)', markersize=8)
plt.plot(model_sizes, [t*1000 for t in model_times_numpy], '--',
         label=f'NumPy model: {k_numpy:.2e}n', linewidth=2)
plt.xlabel('Matrix Size (nn)')
plt.ylabel('Execution Time (ms)')
plt.title('Matrix Multiplication: Measured vs. Mathematical Model')
plt.legend()
plt.grid(True)
plt.show()

Zakljuak

U ovoj laboratorijskoj vjebi, upoznali smo se sa:

  • virtuelnim Python okruenjima koji se kreiraju pomou venv modula kako bi se, izmeu ostaloga, postigla bolja izolacija dostupnih verzija biblioteka izmeu razliitih projekata,
  • NumPy (numpy) bibliotekom za numerika izraunavanja nad n-dimenzionalnim nizovima (vektorima, matricama itd.),
  • matplotlib bibliotekom za plotovanje grafika,
  • tehnikama mjerenja vremena izvravanja pomou time biblioteke i
  • tehnikom izrade matematikog modela (funkcije) kojom se moe predvidjeti vrijeme izvravanja datog algoritma, na koritenom raunaru, za prozivoljnu veliinu ulaza, na osnovu nekoliko mjerenja stvarnog vremena izvravanja.

Uporedili smo performanse mnoenja matrica koristei:

  • Sopstvenu implementaciju.
  • Optimizovanu matmul funkciju dostupnu u NumPy.

Uvidjeli smo sljedee:

  • Naa implementacija ima vremensku sloenost $\Theta(n^3)$, to se jasno odraava na njenom grafiku.
  • NumPy-ev matmul znaajno nadmauje nau implementaciju, esto za nekoliko redova veliine.
  • to vie dimenzije matrica rastu, to je NumPy implementacija znaajnije bra u odnosu na nau sopstvenu implementaciju.
  • Matematiki model $t(n) = Kn^3$ dobro odgovara stvarnim performansama nae implementacije.

NumPy postie ovu prednost u performansama kroz razne optimizacije:

  • Koritenje instrukcija specifinih za procesor koje mogu istovremeno da procesiraju vei broj elemenata.
  • Pristup memoriji optimiziovan za nain funkcionisanja ke memoriju.
  • Optimizovani algoritmi poput Strassen-ovog algoritma za velike matrice.
  • Vienitno izvravanje za velike matrice.
  • Kompajlirani C/C++ kod umjesto interpretiranog Python koda.

Ovime je demonstrirano da je koritenje optimizovanih biblioteka poput NumPy-a kljuno za numeriko izraunavanje u Python-u.
