apstraktne klase se mogu ne inicijalizovati ali mogu imati konstruktor i main ();

i to mora da postoji svaki put pri pozivu konstruktora.};)

externalizable mora da postoji podrazumijevani konstruktor koji mora biti public pri deserijalizaciji (readExternal) i 
poziva se prije read metode (ako ne postoji bacice se izuzetak)

konstruktoru nasljednice, pa onda se inicijalizuju clanovi roditeljske klase klase pa onda se nastavi
sa izvrsavanjem konstruktora.

Prije konstruktora se pozivaju staticki blokovi. oni se pozivaju samo jednom, bez obzira koliko objekata se kreira
